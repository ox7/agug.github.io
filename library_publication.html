发布库工程

默认库工程只对外发布release变体的库。所有引用依赖这个库工程的工程都使用这个变体的库，而不用管到底构建了哪些变体。可以通过控制变体来改变发布的库：

android {
    defaultPublishConfig "debug"
}

注意：当发布配置的名称必须是变体的全名，Release and debug仅没有配置风格的时候可以使用。否则需要加入风格变体名：

android {
    defaultPublishConfig "flavor1Debug"
}

还需要发布所有变体的库，在普通的工程与工程之间依赖时，计划可以使用，但由于Gradle的限制目前这样做还是不合法的。默认发布全变体库是不支持的，打开的话需要如下操作：

android {
    publishNonDefault true
}

多个变体多个aar而不是用一个aar包含多个变体内容这个是非常重要的。这样每个独立的aar只包含一个变体，并且可以作为Gradle项目将aar作为可输出构件。这样不论是发布到maven库还是工程依赖库工程都可以一样使用。

Gradle有一个默认构件的概念，通常如下：

dependencies {
    compile project(':libraries:lib2')
}

当创建一个需要依赖另外变体的发布构件，需要如下使用：

dependencies {
    flavor1Compile project(path: ':lib1', configuration: 'flavor1Release')
    flavor2Compile project(path: ':lib1', configuration: 'flavor2Release')
}

Important: 发布配置是全变体的，包括build type和所有需要的引用。
Important: 当启用非默认变体发布，maven发布插件将会把额外的变体作为扩展包发布。也就是说这样发布不是真正兼容的，应该发布一个独立的变体或者为工程内依赖启用所有配置。
