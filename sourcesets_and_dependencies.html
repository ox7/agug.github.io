Sourcesets and Dependencies

源码和依赖

和Build Type一样，Product Flavors也可以有自己的源码和资源。下面将会创建四个Sourcesets：

  • android.sourceSets.flavor1
    Location src/flavor1/
  • android.sourceSets.flavor2
    Location src/flavor2/
  • android.sourceSets.androidTestFlavor1
    Location src/androidTestFlavor1/
  • android.sourceSets.androidTestFlavor2
    Location src/androidTestFlavor2/

这些Sourcesets都被用于构建APK。在处理所有Sourcesets构建一个APK时遵照如下规则：

  • 所有源码(src/*/java)会以多个文件夹形式一起生成输出
  • Manifest都会被合并到一个独立的Manifest文件中。和Build Type一样Product Flavors也可以声明自己不同的元素和权限。
  • 所有的资源(Android res和assets)都会优先覆盖于Build Type重写Product Flavor的地方。
  • 每个Build Variant都会生成他自己的R class，变体之间无法共享。

最后，和Build Type一样Product Flavors也会有自己的依赖，举个例子：

dependencies {
    flavor1Compile "..."
}

每个变体还可创建额外的sourcesets：

  • android.sourceSets.flavor1Debug
    Location src/flavor1Debug/
  • ndroid.sourceSets.flavor1Release
    Location src/flavor1Release/
  • android.sourceSets.flavor2Debug
    Location src/flavor2Debug/
  • android.sourceSets.flavor2Release
    Location src/flavor2Release/

这些sourcesets会比Build Type的sourcesets又更高的优先级。而且也支持自定义变体等级。
