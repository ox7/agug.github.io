Basics and Configuration

基础和配置

之前提到了，androidTest默认目录路径为src/androidTest/，可以用这个sourceSet构建一个测试APK，而且可以被安装到设备上使用Android testing framework测试应用。主要包括了：unit tests、instrumentation tests、uiautomator tests。
manifest文件中的结点需要创建，并且还可以创建src/androidTest/AndroidManifest.xml用于额外的参数配置。

测试App只很少几个参数可以配置，为了配置节点:

  • testApplicationId
  • testInstrumentationRunner
  • testHandleProfiling
  • testFunctionalTest

如之前介绍，这些可以在defaultConfig类中配置：

android {
    defaultConfig {
        testPackageName "com.test.foo"
        testInstrumentationRunner "android.test.InstrumentationTestRunner"
        testHandleProfiling true 
        testFunctionalTest true
    }
}

测试应用manifest中得instrumentation节点的targetPackage属性值是自动填充测试app的包名的。还可以通过defaultConfig或Build Type自定义。manifest这部分内容也是自动生成的。

另外，androidTest源码也可以配置自己的依赖。默认应用和其依赖都是添加到测试app的classpath中得，但也可以通过这样去扩展：

dependencies {
    androidTestCompile 'com.google.guava:guava:11.0.2'
}

测试app是由assembleAndroidTest任务创建的，它不依赖主的assemble任务。当测试被运行时自动调用创建。

可以通过修改下面配置，更改默认的debug构建类型构建的app：

android {
    ...
    testBuildType "staging"
}
