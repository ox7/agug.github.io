Signing Configurations

签名配置

对一个应用签名需要如下(签名一个APK更多细节可参考Signing Your Application )：

  • 一个keystore
  • keystone密码
  • key 别名
  • key 别名密码
  • store类型

以上这些属性都可以在Signing Configuration查看到具体说明。

默认的，会使用debug配置，debug keystrore路径为$HOME/.android/debug.keystore,，如果没有会自动创建。debug构建类型Build Type 会自动使用这个文件进行debug签名。

通过signingConfigs的DSL容器我们可以自定义这些配置：

android {
    signingConfigs {
        debug {
            storeFile file("debug.keystore")
        }


        myConfig {
            storeFile file("other.keystore")
            storePassword "android"
            keyAlias "androiddebugkey"
            keyPassword "android"
        }
    }


    buildTypes {
        foo {
            signingConfig signingConfigs.myConfig
        }
    }
}

上述构建片段，改变了默认的debug keystore的路径，当debug构建类型会自动使用这个新的路径的签名文件。 还创建了一个新的Signing Config和一个使用它的新的构建类型。

Note: 仅debug keystore会在默认路径中被自动创建。更改debug keystore路径不会自动创建它。创建一个不同名字的SigningConfig，默认的debug keystore会自动创建并使用，换句话说，它和keystore的位置绑定而不是配置的名字。

Note: keystore路径是工程根目录的相对路径，而不能是绝对路径。

Note: 如果你不想在文件中配置密码，那么following Stack Overflow post可以提供从控制台或者环境变量中获取密码的方法。
