Android tasks

Android 任务

Android plugin和其他plugin在使用方式上保持一致，并加入了一些额外的锚点任务：

  • assemble
    装配工程所有输出
  • check
    执行所有检查
  • connectedCheck
    执行是否有已经连接的设备或模拟器检查
  • deviceCheck
    使用APIs连接远程设备检查，在CI集成服务器中很有用
  • build
    执行assemble和check任务
  • clean
    清除工程所有产出

新的锚点任务在没有已连接设备情况下可以被有规则的检查和执行。注意build任务不依赖于deviceCheck和connectedCheck任务。

一个Android工程至少有两个输出：一个debug APK和一个release APK, 它们各自有自己的锚点任务以便于分开执行：

  • assemble
    • assembleDebug
    • assembleRelease

它们都需要依赖那些构建一个APK需要执行的复杂步骤的任务，assemble任务依赖于它们，以便被调用时输出两个APK。

Tip: Gradle支持任务名称的驼峰写法，举个例子：

    gradle aR

和执行:

    gradle assembleRelease

是一样的操作。

其他锚点任务也有它们自己的依赖任务：

  • check
    • lint
  • connectedCheck
    • connectedAndroidTest
  • deviceCheck

最后，插件也会根据构建类型(debug, release, test)创建安装和卸载APK任务：

  • installDebug
  • installRelease
  • uninstallAll
    • uninstallDebug
    • uninstallRelease
    • uninstallDebugAndroidTest
