Product Flavor Configuration

风味配置

每个Flavors都可以用闭包来配置：

android {
    ...


    defaultConfig {
        minSdkVersion 8
        versionCode 10
    }


    productFlavors {
        flavor1 {
            packageName "com.example.flavor1"
            versionCode 20
         }


         flavor2 {
             packageName "com.example.flavor2"
             minSdkVersion 14
         }
    }
}

每一个android.productFlavors.*都是ProductFlavor类型，它们共用相同的属性。

defaultConfig为每个Flavors提供一样的基本配置，当然每个Flavors也可以重写这些配置：

  • flavor1
    • packageName: com.example.flavor1
    • minSdkVersion: 8
    • versionCode: 20
  • flavor2
    • packageName: com.example.flavor2
    • minSdkVersion: 14
    • versionCode: 10

通常来说，Build Type的配置会覆盖其他配置。举个栗子：Build Type的packageNameSuffix会添加到 Product Flavor的packageName中。
